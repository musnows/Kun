# Kun v0.2.30

这一版是 v0.2.29 之后一次围绕模型接入、请求可观测性和多任务协作的更新。Kun 现在可以把多个上游模型组织成一个本地兼容 OpenAI 的路由网关；Provider 设置支持更多订阅来源和模型元数据；Code 对话也有了可查看真实请求链路的「Agent 视角」。同时，会话整理、附件预览、子代理调度和密钥保护都做了进一步完善。

### 模型接入与本地路由

- Settings -> Providers 重新整理为更清晰的供应商、模型与高级本地 Relay 工作区，长配置和高级选项在窄窗口中也可以正常滚动。
- 新增本地模型路由池：可为多个 Provider/模型创建一个稳定的虚拟模型别名，并按优先级、轮询、加权轮询、最低延迟或自适应策略选择上游。
- 路由池会依据目标能力、网络/超时/限流/认证等失败类型和健康状态，在模型尚未输出内容时进行安全切换；路由事件保留在本机，便于检查实际命中的上游。
- 启用本地 Relay 后，Kun Runtime 在回环地址提供兼容 OpenAI 的 `/v1/models`、`/v1/chat/completions` 与 `/v1/responses` 接口，外部本地客户端可使用路由别名而不需要接触上游凭据。
- 模型导入接入 models.dev 元数据，可补充上下文窗口、能力与参数信息；刷新后也会将可用元数据应用到已导入模型。
- 订阅型 Provider 支持添加多个账号配置。新增 Gemini 订阅（通过官方 Antigravity CLI）和 Cursor 订阅（通过官方 Cursor SDK 与 API Key）接入。
- Grok 订阅改为浏览器 OAuth 登录，支持粘贴授权码或回调 URL 完成登录，并会在凭据临近过期或请求被拒绝时自动刷新。Grok Imagine 也加入图片和视频生成能力。

### Agent 视角与请求排查

- Code 对话右侧新增「Agent 视角」，按时间线展示一次对话中的模型请求、工具调用、标题生成与请求构成。
- 对 HTTP 兼容 Provider，可查看最终请求地址、脱敏请求头、请求体、原始响应、流式事件、解析结果、用量和耗时，并可比较传输重试及 `stream_options` 兼容回退。
- 工具按 Kun 系统工具、MCP、扩展和其他来源分组，排查某次请求为何携带特定工具或上下文更直观。
- 请求凭据会在进入记录前脱敏，记录只保存在本机且不会改变、重放或阻塞原始请求。

### 会话、附件与侧边栏

- 回复生成期间加入的排队消息现在会持久化，支持拖拽或键盘调整顺序，并可在发送状态变化后可靠恢复。
- 会话侧边栏支持创建、重命名和删除仅用于整理的目录；可把会话拖入目录，删除目录不会删除会话或项目文件。
- 分支对话可以单独选择草稿模型和 reasoning effort，Code 右侧栏也提供了更直接的创建入口。
- 会话时间线显示回合编号，标题栏和分支对话的视觉层次进一步优化。
- 会话级文件投放区扩大并与 Composer 合并；媒体附件预览改为按需加载和延后渲染，较大附件或较长会话的打开体验更平稳。
- 原有 Todo 面板已移除，右侧区域改为聚焦 Code、变更和 Agent 视角等当前工作流。

### 子代理与指令编排

- 新增可路由的内置 Agent 目录，覆盖代码实现、测试、审查、安全、研究、文档、规划、Write 与 Design 等专长；Kun 会结合任务和当前界面选择合适的角色。
- Design 模式的角色拥有明确的委派档案和指令，可将产品规划、UX 研究、界面实现与设计审查更准确地交给子代理。
- 子代理设置支持批量设置模型和 reasoning effort，并新增「使用已有 Agent」选项；最大并行子代理输入上限提高到 256。
- Agents 设置可配置最大并发回合数，方便在吞吐、资源占用和 Provider 限流之间取舍。
- 系统指令将稳定契约、线程档案和本回合上下文分离，并根据实际宣告的能力生成工具引导，减少不相关上下文和错误工具假设。

### 稳定性与安全

- 模型路由、异步链路测试和运行时热同步的状态展示更准确；会话中发生 Runtime 错误时会提供更清晰的错误呈现。
- 密钥存储会识别现有加密凭据；当操作系统密钥服务暂时不可用时，Kun 会拒绝静默替换加密密钥，避免 OAuth 凭据变得不可恢复。
- Git 检查点改进了新建仓库（尚无首次提交）与恢复场景的处理，并增加了更多一致性校验。
- Agent 请求记录、凭据刷新与模型请求处理补充了异步和边界测试，提升长会话及多 Provider 场景的可靠性。

### 升级说明

- 从 `v0.2.29` 升级可直接通过 GUI 更新；现有工作区、Provider、MCP、Skills 和会话数据无需手动迁移。
- 本地 Relay 只允许在回环地址启用，且对本机客户端不要求 Bearer Token。请先在设置中测试路由池，并为公开给本地客户端的模型别名设置稳定、无冲突的名称。
- 「Agent 视角」会在本机保存请求和响应正文，虽然 URL、请求头和已知凭据会脱敏，但其中仍可能包含提示词、代码和模型输出。分享诊断文件前请自行审阅内容。
- Grok 订阅不再提供设备码登录入口。如浏览器未能自动回调，请将 Grok 页面显示的授权码或完整回调 URL 粘贴回 Kun 完成登录。
- 如系统钥匙串、Windows DPAPI 或 Linux Secret Service 暂时不可用，Kun 可能提示无法安全加载既有凭据；恢复系统凭据服务后重试，不要删除 Kun 的密钥文件或凭据目录。

### 总结

v0.2.30 的重点是让 Kun 在多模型、多会话和多子代理的真实工作流里更可控：本地 Relay 将多个上游配额收束为稳定入口，Agent 视角让每次模型调用更容易解释，会话目录、队列和附件让日常操作更顺手，而订阅接入与密钥保护则让连接外部模型时更稳妥。
